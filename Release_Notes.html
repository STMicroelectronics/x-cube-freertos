Release Notes for X-CUBE-FREERTOS

Copyright © 2023 STMicroelectronics

Purpose

With FreeRTOS complementing the extensive STM32Cube ecosystem providing free development tools, software bricks, and software expansion packages, STM32 users can also leverage the rich services of FreeRTOS which meet the needs of tiny, smart, connected devices.

X-CUBE-FREERTOS (FreeRTOS Software Expansion for STM32Cube) provides a full integration of FreeRTOS in the STM32Cube environment for the STM32U5, STM32H5, STM32WBA, STM32C0, STM32U0, STM32U3 and STM32N6 series of microcontrollers. Ready-to-run applicative examples are also provided for the NUCLEO-H563ZI, NUCLEO-U575ZI-Q, NUCLEO-WBA52CG, NUCLEO-C031C6, NUCLEO-U083RC, NUCLEO-U385RG-Q and NUCLEO-N657X0-Q thus reducing the learning curve and ensuring a smooth application development experience with FreeRTOS and STM32U5, STM32H5, STM32WBA, STM32C0, STM32U0, STM32U3 and STM32N6 MCUs.

Repository structure

The STMicroelectronics X-CUBE-FREERTOS repository consists of the following repositories:

  • STM32CubeMX: contains STM32CubeMX configuration files:
    • XML files: they describe how the software component configuration parameters can be modified through the STM32CubeMX user interface
    • FTL files: they provide the initialization code to be generated by the STM32CubeMX
  • Drivers: contains STM32 CMSIS and HAL drivers.
  • Middlewares: contains FreeRTOS Middleware files.

Known limitations

  • When generating projects using the STM32CubeMX tool for STM32N6 projects, the following manual steps and considerations are required :

    • FreeRTOS function calls(Init Scheduler and Start Scheduler) must be manually added to main.c for both “secure domain only” and “Secure and Non-Secure domains”
    • The IAC_IRQHandler must be manually added to AppliSecure/Src/stm32n6xx_it.c file for “Secure and Non-Secure domain”
    • RIF settings must be manually added to the AppliSecure/Src/main.c file for “Secure and Non-Secure domains” (refer to the FreeRTOS_Queues_ThreadFlags_TrustZone application)
    • In the AppliNonSecure project, the SysTick IRQ handler should be unchecked in the NVIC configuration tab within STM32CubeMX UI for “Secure and Non-Secure Domains”
    • The partition_stm32n657xx.h file lacks a designated user code section. As a result, any custom configurations are lost upon example project re-generation for “Secure and Non-Secure domains”
    • After re-generation the FreeRTOS_Queues_ThreadFlags_TrustZone application, STM32CubeMX may generate incorrect Secure and Non-Secure project names. These must be manually deleted

  • The following projects are not provided with the STM32CubeIDE:

    • NUCLEO-N657X0-Q/Applications/FreeRTOS_Queues_ThreadFlags_TrustZone
    • NUCLEO-N657X0-Q/Applications/FreeRTOS_MPU

  • The FreeRTOS_MPU application in NUCLEO-N657X0-Q is functional when the optimization level of the mpu_wrapper.c file is set to “-O2” with the MDK-ARM toolchain.

  • The FreeRTOS_Semaphore_LowPower application in NUCLEO-U575ZI-Q is functional when the power cable is unplugged and plugged back into the board.

  • When generating project with TrustZone activated, end user needs to:

    • Activate the ‘CMSIS RTOS2/CORE’ component in both secure and non-secure contexts.
    • Add SysTick_Handler IRQ handler to Secure/Src/stm32u5xx_it.c/stm32h5xx_it.c/stm32u3xx_it.c files.

Known limitations

  • The X-CUBE-FREERTOS pack does not support the CM55, such as the STM32N6 family.

  • The STM32N6 projects are not generated with STM32CubeMX tool.

  • In the case of creating a project from scratch for the CM55, such as the STM32N6, or the CM33, such as the STM32U5, STM32H5, STM32WBA, and STM32U3, the user must include the path ‘…/FreeRTOS/Source/include’ in the Assembler-Preprocessor-Additional include directories option within EWARM projects.

  • The following projects are not supported with the STM32CubeIDE toolchain:

    • NUCLEO-N657X0-Q/Applications/FreeRTOS_Queues_ThreadFlags_TrustZone
    • NUCLEO-N657X0-Q/Applications/FreeRTOS_MPU

  • The FreeRTOS_MPU application in NUCLEO-N657X0-Q is functional when the optimization level of the mpu_wrapper.c file is set to “-O2” with the MDK-ARM toolchain.

  • The FreeRTOS_Semaphore_LowPower application in NUCLEO-U575ZI-Q is functional when the power cable is unplugged and plugged back into the board.

  • When generating project with TrustZone activated, end user needs to:

    • Activate the ‘CMSIS RTOS2/CORE’ component in both secure and non-secure contexts.
    • Add SysTick_Handler IRQ handler to Secure/Src/stm32u5xx_it.c/stm32h5xx_it.c/stm32u3xx_it.c files.
